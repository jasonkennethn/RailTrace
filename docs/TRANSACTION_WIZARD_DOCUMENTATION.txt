# Transaction Wizard Component Documentation

## Overview
The TransactionWizard component is a mobile-first, wizard-style interface for displaying blockchain transaction details, following the exact Stitch AI design approach from txs.html. It provides a clean, professional, and government-app-friendly interface for viewing detailed transaction information.

## Key Features

### 1. Mobile-First Design
- **Responsive Layout**: Single-column, stacked layout optimized for mobile devices
- **Touch-Friendly**: Minimum 44px touch targets for all interactive elements
- **Full-Width Cards**: Cards utilize full screen width on mobile for better readability
- **Smooth Animations**: Slide and fade transitions for step expansions

### 2. Wizard-Style Step Expansion
- **Progressive Disclosure**: Extra data is hidden by default, expands step-by-step
- **Four Main Steps**:
  - Basic Information (Part ID, Date/Time, Status)
  - Transaction Details (Hash, Block Number, Event Type)
  - Part Information (Part Hash, Condition)
  - Location & Personnel (Depot ID, Officer ID, Location)
- **Smooth Transitions**: 300ms ease-in-out animations for expanding/collapsing

### 3. Data Display & Interaction
- **Smart Hash Truncation**: Shows first 4 + last 4 characters (e.g., 0x32d6...6edb46)
- **Copy-to-Clipboard**: Individual copy buttons for hashes and important data
- **Toast Feedback**: Instant visual feedback when copying data
- **Icon Integration**: Material Symbols icons for each field type

### 4. Visual Design
- **Glass Morphism**: Subtle backdrop blur effects for modern appearance
- **Rounded Corners**: 12-16px border radius for cards
- **Stitch AI Colors**: Primary blue (#1773cf) with proper contrast ratios
- **Typography Hierarchy**: Clear heading/body text distinction
- **Status Indicators**: Color-coded status badges with dots

### 5. Action Buttons
- **View on BSCScan**: Full-width primary button with external link icon
- **Copy Details**: Secondary button for copying all transaction data
- **Back Navigation**: Proper back button with smooth navigation

## Component Structure

### TransactionWizard.tsx
```typescript
interface TransactionData {
  partId: string;
  partHash: string;
  transactionHash: string;
  blockNumber: number;
  eventType: string;
  depotId: string;
  officerId: string;
  location: string;
  condition: string;
  dateTime: string;
  status: 'confirmed' | 'pending' | 'failed';
  additionalData?: Record<string, any>;
}

interface WizardStep {
  id: string;
  title: string;
  icon: React.ComponentType<{ className?: string }>;
  fields: Array<{
    key: string;
    label: string;
    value: string | number;
    icon: React.ComponentType<{ className?: string }>;
    copyable?: boolean;
  }>;
}
```

### Key Functions
- `copyToClipboard()`: Handles clipboard operations with error handling
- `toggleStep()`: Manages step expansion/collapse state
- `truncateHash()`: Smart hash truncation for mobile display
- `getStatusConfig()`: Returns appropriate styling for transaction status
